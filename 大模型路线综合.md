# 大模型路线综合

> 来源：`路线/6 个月大模型算法校招上岸_3` 到 `_29` 图片内容整理。  
> 目标：把图片里的 6 个月大模型算法校招路线，整理成可执行的 Markdown 计划。

## 总目标

这条路线的核心不是“泛泛学大模型”，而是在 6 个月内建立一套可以被面试验证的能力链：

- 代码能力：LeetCode 200 题，覆盖热题、高频题、中等题，常见 loss 函数能写伪代码。
- 面试能力：完成 60 次模拟问答，覆盖技术、项目、HR。
- 投递结果：100-180 次有效投递，拿到 20+ 面试，目标 3-8 个 offer，包含实习或正式岗位。

## 执行原则

必须遵守这些原则，否则容易学散、做虚、面试讲不出来：

1. 每周至少投入 25 小时。  
   在职可以压到 18-20 小时，但周期要延长。

2. 每周必须有“可展示产出”。  
   产出可以是代码、文档、复盘、博客、视频之一。

3. 每周日固定复盘。  
   复盘格式：输入 -> 输出 -> 问题 -> 下周修正。

4. 项目优先级高于刷题数量，刷题数量高于知识广度。  
   简历和面试最终看的是你能不能把项目讲深，而不是看了多少零散资料。

5. 任何知识点都要能回答四件事：定义、原理、工程取舍、线上问题。

## 第 1 个月：打基础 + 建知识框架

### 月目标

1. 补齐数学与机器学习基础知识。
2. 读懂 Transformer 核心代码与训练基本流程。
3. 建立“面试知识库”和“错题本”。

### 第 1 周：Python + PyTorch 基础巩固

任务：

- Python + PyTorch 基础巩固：Tensor 操作、自动求导、Dataloader。
- 手写：线性回归、MLP、训练循环，包含验证集。
- 输出：1 份训练模板仓库，放到 GitHub。

验收标准：

- 能解释 Tensor、梯度、反向传播、优化器的基本关系。
- 能独立写出一个标准 PyTorch 训练脚本。
- 能解释每段训练代码为什么这么写。

### 第 2 周：机器学习与概率统计高频

任务：

- 机器学习高频：逻辑回归、SVM、树模型、偏差方差、正则化。
- 概率统计高频：期望、方差、MLE、贝叶斯、假设检验。
- 输出：30 个面试 Q/A 速记卡。

建议整理的问题：

- 逻辑回归和线性回归有什么区别？
- L1 和 L2 正则有什么区别？
- SVM 的间隔最大化怎么理解？
- 决策树、GBDT、随机森林、XGBoost 的区别是什么？
- MLE 和 MAP 的区别是什么？
- 假设检验里的 p-value 怎么解释？

### 第 3 周：Transformer 全链路

任务：

- Transformer 全链路：Embedding、MHA、FFN、Norm、Residual。
- 重点搞懂：为什么要多头注意力、Pre-LN vs Post-LN、RoPE、KV Cache。
- 输出：1 篇技术长文，可以作为简历附件或博客链接。

验收标准：

- 能从零讲清 Transformer 每个模块的作用和复杂度。
- 能解释多头注意力为什么有用。
- 能说清 Pre-LN 和 Post-LN 的差异。
- 能解释 RoPE 和 KV Cache 的基本思想。

### 第 4 周：训练基础与第一次模拟面试

任务：

- 训练基础：损失函数、优化器、学习率策略、梯度裁剪、混合精度。
- 常见问题排查：过拟合、梯度爆炸、训练不收敛。
- 输出：第一次模拟面试，可以找现成面试速记 Q/A 帖。

验收标准：

- 能讲清常见 loss 的适用场景。
- 能解释训练不收敛的排查路径。
- 能写出标准 PyTorch 训练脚本并解释每段代码。
- 完成简历初稿 v1，即使项目还少也先写一版。

## 第 2 个月：大模型训练与对齐

### 月目标

1. 掌握 SFT / RLHF / DPO / GRPO 的原理与差异。
2. 掌握参数高效微调：LoRA / QLoRA。
3. 能讲清训练数据构造与评估方法。

### 第 1 周：SFT 数据与实操

任务：

- 学习 SFT 数据格式、指令构造、清洗与去重。
- 用开源小模型完成一次 SFT。
- 输出：训练配置文档 + 结果对比表。

重点：

- 训练数据不要只会说“网上找的”，要讲清数据来源、清洗规则、去重规则和数量变化。
- 要能解释 instruction、input、output 格式。
- 要能展示训练前后回答差异。

### 第 2 周：DPO / PPO / GRPO 与偏好数据

任务：

- 学习 DPO / PPO / GRPO：目标函数、优缺点、适用场景。
- 准备高频追问：为什么不用纯 SFT、偏好数据怎么构建。
- 输出：20 个对齐面试题标准答案。

需要能回答：

- SFT、RLHF、DPO 的关系是什么？
- DPO 为什么不需要显式 reward model？
- PPO 为什么训练复杂？
- 偏好数据如何构建，质量差会怎样？
- beta 参数调大调小有什么影响？

### 第 3 周：LoRA / QLoRA 与训练成本优化

任务：

- 学习 LoRA / QLoRA：rank、target_modules、量化取舍。
- 学习训练成本优化：batch、gradient accumulation、checkpoint、flash attention。
- 输出：至少 3 组微调实验记录。

建议实验：

- rank=4 / 16 / 64 的效果与显存对比。
- `q_proj+v_proj` 与全线性层 target_modules 对比。
- LoRA 与 QLoRA 显存、速度、效果对比。

### 第 4 周：模型评估与项目一完成

任务：

- 模型评估：困惑度、任务指标、人工评测、A/B。
- 失败案例复盘：复读机、风格漂移、泛化下降。
- 输出：项目一完成，可以叫“小模型指令微调与评估”，也推荐自己找开源项目跟着做。

验收标准：

- 能把 SFT -> DPO -> RLHF 讲成“能力注入 -> 偏好对齐 -> 策略优化”。
- 简历上有第一个可深挖项目，且包含指标前后对比。
- 项目可复现：README、脚本、配置齐全。

## 第 3 个月：RAG 与检索系统

### 月目标

1. 完整掌握 RAG：切块、召回、重排、生成、评测。
2. 能独立搭建可演示的 RAG 系统。
3. 能回答线上故障与修复策略。

### 第 1 周：文档处理与向量库实操

任务：

- 文档处理：清洗、分块、元数据、索引构建。
- 向量库实操：Milvus 或 FAISS 任选一个。
- 输出：数据处理 pipeline。

重点：

- 能解释 chunk_size、overlap 怎么选。
- 能解释 PDF / 多栏 / 表格为什么不能直接按纯文本切块。
- 能保存 metadata，例如文档来源、页码、段落 ID。

### 第 2 周：召回策略与评测

任务：

- 召回策略：向量召回 + BM25 混合召回。
- query 改写、多路召回、去重融合。
- 输出：召回评测脚本，核心指标是 Recall@K。

建议实验：

- 纯向量召回 vs BM25 vs 混合召回。
- TopK 取 3 / 5 / 10 的效果对比。
- 不同 chunk_size 对 Recall@K 的影响。

### 第 3 周：重排、生成约束与 Demo

任务：

- 重排模型：cross-encoder 或规则重排。
- 生成约束：证据绑定、拒答机制、引用溯源。
- 输出：RAG v1 在线 Demo。

重点：

- 低置信度时拒答，不要硬编。
- 回答里要能引用来源。
- 要能演示“输入问题 -> 召回文档 -> 重排 -> 生成回答”的完整链路。

### 第 4 周：评测体系与项目二完成

任务：

- 建立评测体系：离线 + 在线，badcase 池与迭代机制。
- 整理典型故障：空召回、答非所问、幻觉、延迟高。
- 输出：项目二完成，可以叫“企业知识库 RAG 助手”，也可以自己找 notebook 或开源项目复刻。

验收标准：

- 能用“问题 -> 根因 -> 修复 -> 指标”讲 RAG 故障。
- 项目二可以现场演示，且有评测数据。
- 写出项目深挖文档，面试问到就直接按文档讲。

## 第 4 个月：推理部署与工程化

### 月目标

1. 掌握推理加速与服务化部署核心。
2. 了解 vLLM / TensorRT-LLM / 量化常见路线。
3. 形成“可上线思维”：吞吐、延迟、稳定性、可观测。

### 第 1 周：推理优化与压测

任务：

- 推理优化：KV Cache、continuous batching、并发控制。
- 量化：INT8 / 4bit 基础与效果取舍。
- 输出：推理压测报告，包含 QPS、P95、显存。

建议记录：

- 不同并发下的 QPS。
- P50 / P95 / P99 延迟。
- 显存峰值。
- max_seq_len 对显存的影响。

### 第 2 周：服务框架与 API

任务：

- 服务框架：FastAPI + 推理后端 + Redis 缓存。
- 接口限流、超时重试、降级策略。
- 输出：可部署 API 服务。

服务链路：

```text
用户请求 -> API 服务 -> 缓存命中则直接返回 -> 未命中走检索/推理 -> 返回结果 -> 写入缓存
```

### 第 3 周：监控告警与异常排查

任务：

- 监控告警：日志、指标、追踪。
- 异常排查：OOM、超时、请求抖动。
- 输出：故障演练文档。

常见故障：

- OOM：查显存、batch、seq_len。
- 延迟飙升：查并发、KV Cache、生成长度。
- 请求抖动：查缓存、GC、显存碎片、批处理策略。
- 输出截断：查 `max_new_tokens` 和流式输出。

### 第 4 周：项目整合

任务：

- 项目整合：训练 / 检索 / 推理串成完整链路。
- 输出：项目三完成，可以叫“可部署 LLM 应用系统”。

验收标准：

- 项目三具备工程化特征，不是纯 Notebook。
- 能讲清性能瓶颈与优化顺序。
- 简历升级到 v2：突出系统能力和指标。

## 第 5 个月：集中求职月

### 月目标

1. 完成定向投递：岗位 -> 公司 -> 部门 -> 优先级。
2. 技术面题库与项目面题库全部成型。
3. 拿到第一批面试，并边面边学、快速迭代。

### 第 1 周：简历冲刺

任务：

- 准备 3 个版本简历：算法向、RAG 向、工程向。
- 每个项目都写出：背景、目标、方案、指标、挑战、复盘。
- 输出：作品集主页。

作品集主页建议包含：

- 简历 PDF。
- 三个项目仓库链接。
- 项目说明文档。
- 技术博客或长文。
- 面试题库目录。

### 第 2 周：投递启动

任务：

- 每天投递 6-10 个岗位，要求精准匹配，不海投垃圾岗。
- 同步找内推：同学、校友、脉脉、技术社区。
- 输出：投递跟踪表，包含公司、岗位、状态、反馈、下一步。

投递表建议字段：

| 公司 | 岗位 | 方向 | 投递日期 | 渠道 | 状态 | 反馈 | 下一步 |
| --- | --- | --- | --- | --- | --- | --- | --- |
|  |  | 算法 / RAG / 工程 |  | 官网 / 内推 / 猎头 |  |  |  |

### 第 3 周：面试强化

任务：

- 每天 1 道手撕 + 1 轮项目问答。
- 高频问题录音复盘，修“废话”和“逻辑断点”。
- 输出：个人面试话术库。

面试话术库至少包含：

- 1 分钟自我介绍。
- 3 分钟自我介绍。
- 每个项目的 2 分钟讲法。
- 每个项目的 5 分钟深挖讲法。
- 高频追问脚本，包含技术 + 项目。

### 第 4 周：复盘修正

任务：

- 按挂面原因分桶：基础薄弱、项目不深、表达混乱。
- 针对性补强，而不是盲目继续投。
- 输出：简历 v3 + 项目讲解 v2。

验收标准：

- 至少拿到 8-15 次面试机会。
- 每次面试后 24 小时内完成复盘。
- 形成“面试追问树”，面试官问到哪都不慌。

## 第 6 个月：冲刺拿 offer

### 月目标

1. 多轮面试并行推进。
2. 提升通过率和谈薪能力。
3. 拿到多家 offer，并做最优选择。

### 第 1 周：面试前准备模板

任务：

- 准备面试前模板：公司业务、JD 匹配、项目映射。
- 自我介绍控制在 60 秒和 180 秒两个版本。

准备方式：

- 每家公司先看业务方向，再把自己的项目映射到岗位需求。
- 对算法岗强调模型、训练、指标。
- 对 RAG 岗强调检索、评测、线上故障。
- 对工程岗强调部署、压测、服务化。

### 第 2 周：系统设计与手撕补盲

任务：

- 系统设计题强化：RAG 系统设计、模型服务架构、评测闭环。
- 手撕补盲：链表、二叉树、DP、滑窗、堆。

要能讲：

- 一个企业知识库 RAG 系统怎么设计。
- 一个模型服务怎么保证低延迟和高可用。
- 如何构建评测集和 badcase 池。
- 如何从日志定位线上问题。

### 第 3 周：压力追问与 HR 面

任务：

- 重点攻克压力追问：为什么这么做、还有什么方案、线上翻车怎么办。
- HR 面准备：动机、稳定性、抗压、职业规划。
- 输出：HR 问答稿。

常见压力问题：

- 为什么这个方案比另一个方案好？
- 如果线上出现幻觉怎么办？
- 如果召回为空怎么办？
- 如果 GPU OOM 怎么办？
- 如果 P95 延迟突然升高，怎么排查？
- 为什么你适合这个岗位？

### 第 4 周：offer 对比与最终选择

任务：

- offer 对比：部门业务、成长空间、导师、技术栈、强度、薪资。
- 谈薪策略：先比较后报价，不先亮底牌。
- 输出：offer 决策表。

验收标准：

- 至少 3 个可选 offer，实习或校招都算。
- 拿到目标公司或同梯队公司录用。
- 完成最终复盘，为入职后 3 个月做准备。

## 每天 / 每周执行模板

### 工作日：2-3 小时

1. 60 分钟：知识学习，只学和岗位相关内容。
2. 60 分钟：项目推进，必须有代码提交。
3. 30 分钟：面试问答，背自己的 Q/A 速记卡、找帖子或手撕。

### 周末：每天 4-6 小时

1. 半天做项目，半天做面试模拟。
2. 周日晚上固定复盘，并排下周计划。

### 每周最小交付

- 代码提交 >= 5 次。
- 技术文档 >= 1 份。
- 模拟面试 >= 2 次。
- 新增题卡 >= 20 条。

## 求职材料清单

必须齐：

1. 简历：算法向 / 工程向 / RAG 向三版。
2. 项目仓库：3 个项目都可运行。
3. 项目说明文档：每个项目至少 1 页。
4. 面试题库：按主题分类。
5. 投递跟踪表：公司、岗位、状态、下一步。
6. 自我介绍脚本：1 分钟、3 分钟。
7. 常见追问脚本：技术 + 项目。

材料准备到这个程度，求职才不只是“投简历”，而是有组织地推进面试漏斗。

## 这么做就等死了

这些坑要主动避开：

1. 只背八股，不做项目。  
   面试第二轮基本就挂。

2. 只做项目，不会讲指标。  
   很难过面评。

3. 刷题只刷简单题。  
   手撕轮容易卡死。

4. 海投不复盘。  
   面试机会会被浪费。

5. 项目无数据对比。  
   缺乏说服力。

6. 回答太空泛。  
   没有“场景 -> 动作 -> 结果”的闭环。

## 最终建议

1. 你不是在“学知识”，你是在“构建可被录用的证据链”。

2. 每个能力都要有对应证据：代码、指标、文档、复盘。

3. 面试本质是“可验证的价值表达”，不是背答案比赛。

4. 连续 6 个月稳定输出，远比偶尔高强度更有用。

执行这套计划，6 个月后不一定百分百拿 offer，但拿不到也至少会形成完整的基础、项目、表达、投递和复盘体系。对大模型算法校招来说，这已经比“只看课、只背题、只收藏路线”强太多。
