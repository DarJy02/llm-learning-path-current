# 数据与参考项目

## 主参考项目

### Open-R1

链接：

- https://github.com/huggingface/open-r1

用途：

- 参考 SFT 脚本。
- 参考 GRPO/RLVR 训练。
- 参考数学 reward 设计。
- 参考 Qwen 模型训练配置。

关键信息：

- 项目目标是复现 DeepSeek-R1 路线。
- 仓库中包含 `sft.py`、`grpo.py`、`generate.py`。
- 支持 SFT 和 GRPO。
- 官方示例里可以把模型替换成 `Qwen/Qwen3-0.6B-Base`。

### AIMO2 第二名方案

链接：

- https://github.com/imagination-research/aimo2

用途：

- 参考真实 Kaggle 数学竞赛方案。
- 参考 SFT + DPO 训练。
- 参考 DPO 数据筛选。
- 参考推理时多采样、自一致投票、早停策略。

关键信息：

- 方案分三部分：推理训练、效率优化、推理策略。
- 推理训练包含 Stage 1 SFT 和 Stage 2 DPO。
- DPO 数据来自 `OpenR1-Math-220k`。
- 他们用 DPO 主要是为了减少输出长度。

### Numina AIMO 第一阶段冠军方案

链接：

- https://github.com/project-numina/aimo-progress-prize

用途：

- 参考 CoT SFT。
- 参考 Tool-Integrated Reasoning，即 TIR。
- 参考 Kaggle 约束下的推理方案。

关键信息：

- 训练分两阶段：NuminaMath-CoT SFT 和 NuminaMath-TIR SFT。
- TIR 数据让模型学会用代码辅助数学推理。
- 原方案使用 8 x 80GB GPU 训练完整模型。

## 主数据集

### OpenR1-Math-220k

链接：

- https://huggingface.co/datasets/open-r1/OpenR1-Math-220k

用途：

- SFT 主数据。
- DPO 数据构造来源。

特点：

- 约 220k 数学问题。
- 每题有 2 到 4 条 DeepSeek-R1 生成的 reasoning trace。
- 大多数样本经过 `Math Verify` 验证。
- 每个问题至少有一条正确 reasoning trace。
- Hugging Face 页面上有 3 个 subset/config：`default`、`extended`、`all`。
- 只有 1 个 split：`train`。
- `default/train` 约 94k，适合作为第一版主数据。

建议用法：

- 第一版只用 `default/train`。
- SFT 选择正确 trace。
- DPO 从同题多 trace 中筛 chosen/rejected。

### NuminaMath-TIR

链接：

- https://huggingface.co/datasets/AI-MO/NuminaMath-TIR

用途：

- 可选第二阶段数据。
- 训练模型用代码辅助解数学题。

特点：

- 约 70k 工具辅助推理样本。
- 适合训练 TIR，即先推理、再写代码验证或计算。

建议用法：

- 不放进第一版。
- 等 SFT + DPO 跑通后，再作为扩展方向。

### MATH-500

链接：

- https://huggingface.co/datasets/HuggingFaceH4/MATH-500

用途：

- 评估集。

建议指标：

- pass@1。
- `\boxed{}` 提取成功率。
- 最终答案正确率。
- 平均输出 token 数。
- 超长输出比例。

### AIME / AIMO 风格题

链接：

- Kaggle AIMO: https://www.kaggle.com/competitions/ai-mathematical-olympiad-prize
- AIMO2 方案内置 AIME 2025 / reference set: https://github.com/imagination-research/aimo2

用途：

- 小规模高难度评估。
- 观察自一致投票效果。

够。

| 阶段 | 数据 |
|---|---|

| 扩展 | `NuminaMath-TIR` |

## 下一步数据处理任务

1. 下载 `OpenR1-Math-220k/default`。
2. 观察字段结构。
3. 抽取正确 reasoning trace 生成 SFT 数据。
4. 按正确性、长度、格式构造 DPO pair。
5. 下载 MATH-500 做评估脚本。
6. 记录每次实验的模型、数据规模、训练阶段和评估结果。
