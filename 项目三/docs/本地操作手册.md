# 本地操作手册：Qwen-Math 数学推理 SFT + DPO

## 当前确定方案

课题赛题：

> Kaggle AIMO / AIMO2 数学推理赛题复现与改造。

技术路线：

> 参考 AIMO2 第二名方案，先做 SFT，再做 DPO。第一版暂时不做 GRPO。

本地模型：

```text
Qwen/Qwen2.5-Math-1.5B-Instruct
```

第一版目标：

```text
原模型 baseline
-> SFT
-> SFT + DPO
-> 对比正确率、输出长度、boxed 答案格式、无法解析比例
```

## 为什么先用 Qwen-Math

普通 Qwen 从头学数学推理，想明显提升正确率会更吃数据和算力。Qwen-Math 已经有数学底子，本地做 SFT/DPO 更稳。

第一版不要追求“模型数学能力暴涨”，更合理的观察目标是：

- SFT 后推理格式更稳定。
- SFT 后 `\boxed{}` 最终答案格式更稳定。
- SFT 后正确率可能小幅提升。
- DPO 后输出长度下降。
- DPO 后答案可解析率提高。
- DPO 后正确率尽量不掉。

这正好贴近 AIMO2 第二名方案中 DPO 的作用：在比赛时间限制下减少过长输出，提高效率和稳定性。

## Step 0：进入项目

```powershell
cd "D:\project\others\1大模型路线继续学习"
```

如果你之前已经能跑 LLaMA-Factory，可以直接复用原环境。

如果环境还没激活，按你之前跑通 LLaMA-Factory 的方式激活即可。

## Step 1：安装数据处理依赖

在 LLaMA-Factory 环境里安装：

```powershell
pip install -U datasets huggingface_hub
```

如果 Hugging Face 下载慢，可以使用镜像：

```powershell
$env:HF_ENDPOINT="https://hf-mirror.com"
```

## Step 2：先不要下载全量，先抽小数据

第一版只抽：

```text
SFT: 1000 条
DPO: 1000 对
```

运行：

```powershell
python "项目三/scripts/prepare_openr1_math.py" `
  --dataset open-r1/OpenR1-Math-220k `
  --subset default `
  --split train `
  --max-sft 1000 `
  --max-dpo 1000 `
  --sft-out "项目三/data/openr1_math_sft_1k.json" `
  --dpo-out "项目三/data/openr1_math_dpo_1k.json"
```

如果脚本发现数据字段和预期不完全一致，会先打印字段信息。到时根据输出微调脚本即可。

## Step 3：把数据放到 LLaMA-Factory

推荐方式：复制到 LLaMA-Factory 的 `data` 目录。

```powershell
Copy-Item "项目三/data/openr1_math_sft_1k.json" "LLaMA-Factory/data/openr1_math_sft_1k.json" -Force
Copy-Item "项目三/data/openr1_math_dpo_1k.json" "LLaMA-Factory/data/openr1_math_dpo_1k.json" -Force
```

然后把 `项目三/configs/llamafactory/dataset_info_patch.json` 里的内容合并到：

```text
LLaMA-Factory/data/dataset_info.json
```

注意：是合并 JSON 条目，不是整个文件覆盖。

需要新增的两个数据集名字是：

```text
openr1_math_sft_1k
openr1_math_dpo_1k
```

## Step 4：原模型 baseline

训练前必须先跑原模型，否则看不出提升。

第一版可以先手动抽 20-50 道 MATH-500 / AIME 风格题，用同一套 prompt 对比：

```text
Please reason step by step, and put your final answer within \boxed{}.

{problem}
```

至少记录：

- 是否答对。
- 是否有 `\boxed{}`。
- 输出 token/字符长度。
- 是否无法解析。

后面再写自动评估脚本。

## Step 5：跑 SFT

进入 LLaMA-Factory：

```powershell
cd "D:\project\others\1大模型路线继续学习\LLaMA-Factory"
```

运行：

```powershell
llamafactory-cli train "../项目三/configs/llamafactory/qwen25_math_1_5b_lora_sft.yaml"
```

如果你的环境里没有 `llamafactory-cli`，用：

```powershell
python src/train.py "../项目三/configs/llamafactory/qwen25_math_1_5b_lora_sft.yaml"
```

SFT 输出目录：

```text
LLaMA-Factory/saves/qwen25-math-1.5b/lora/aimo_sft_1k
```

## Step 6：跑 SFT 后评估

用和 baseline 完全相同的题、prompt、采样参数再跑一遍。

对比：

```text
原模型 vs SFT 模型
```

重点看：

- 正确率有没有提升。
- `\boxed{}` 格式是否更稳定。
- 输出是否明显变长。

## Step 7：跑 DPO

DPO 要接在 SFT adapter 后面跑。配置文件里已经写了：

```yaml
adapter_name_or_path: saves/qwen25-math-1.5b/lora/aimo_sft_1k
```

运行：

```powershell
llamafactory-cli train "../项目三/configs/llamafactory/qwen25_math_1_5b_lora_dpo.yaml"
```

或：

```powershell
python src/train.py "../项目三/configs/llamafactory/qwen25_math_1_5b_lora_dpo.yaml"
```

DPO 输出目录：


## 第一版是否算成功

不要只用“正确率大涨”判断成功。

第一版只要出现下面任意两三项，就算项目立住了：

- SFT 后 boxed 格式明显更稳定。
- SFT 后正确率小幅提升。
- SFT 后推理过程更完整。
- DPO 后平均输出长度下降。
- DPO 后无法解析率下降。
- DPO 后正确率没有明显下降。
