# 项目二：Text-to-SQL 大模型高效微调与执行评测

## 当前定位

项目二从电商客服方向切换到 Text-to-SQL，目标是保留项目一的完整方法论，但换成一个数据更充足、评测更客观、业务落地更强的任务。

一句话目标：

> 基于 Qwen2.5-Coder-7B-Instruct，对 Text-to-SQL 任务进行 QLoRA SFT 微调，并通过 SQL 合法率、字段/表名命中、执行正确率和 badcase 分析评估模型能力。

## 为什么切换方向

项目一的电商客服公开数据存在明显问题：

1. 客服流水数据模板句多、寒暄多、低信息轮次多。
2. 回答质量主观，自动评测很难说明模型真的变好。
3. 即使用更大参数模型，也难以从低质量数据中学出稳定高质量回答。
4. 目前最多能做到“及时停止、少复读”，但业务回答质量仍然勉强。

因此项目二不再做电商客服，而改做 Text-to-SQL。

## 为什么选 Text-to-SQL

Text-to-SQL 的业务落地场景明确：

```text
自然语言查数
BI 数据分析助手
运营/销售/财务自助取数
企业数据问答
数据库智能助手
```

它比客服项目更适合做简历项目，因为：

1. 数据集充足，有 Spider、BIRD、WikiSQL、text-to-sql-mix-v2 等公开数据。
2. 评测更客观，可以看 SQL 是否可解析、是否可执行、执行结果是否正确。
3. badcase 好分类：表名错、字段错、JOIN 错、WHERE 条件漏、聚合错、排序错。
4. 后续可以自然接 DPO：正确 SQL 作为 chosen，常见错误 SQL 作为 rejected。

## 当前推荐模型

主线模型：

```text
Qwen/Qwen2.5-Coder-7B-Instruct
```

原因：

1. 参数量大于 Qwen3-4B，项目观感更强。
2. Coder 模型更适合结构化代码/SQL 生成。
3. T4 16G 可以通过 4bit QLoRA 训练。
4. 比 Qwen3 普通模型更贴 Text-to-SQL。

训练方式：

```text
4bit QLoRA
LoRA SFT
cutoff_len 1024 或 1536
batch_size 1
gradient_accumulation 16
```

## 当前推荐数据

优先级：

1. `text-to-sql-mix-v2`
   - 混合 Spider、BIRD、WikiSQL 等来源。
   - 数据量大，适合做 SFT。
   - 地址：https://huggingface.co/datasets/DanielRegaladoCardoso/text-to-sql-mix-v2

2. BIRD
   - 更贴真实数据库问答。
   - 难度更高，适合作为后续评测或进阶数据。

3. Spider
   - 经典 Text-to-SQL benchmark。
   - 适合固定评测集和论文式实验说明。

4. WikiSQL
   - 相对简单。
   - 适合早期跑通流程。

## 项目主线

```text
公开 Text-to-SQL 数据
-> 数据格式分析
-> 转成 LLaMA-Factory SFT 格式
-> Qwen2.5-Coder-7B-Instruct + QLoRA SFT
-> 固定评测集生成 SQL
-> SQL 合法率 / 执行正确率 / badcase 分析
-> 构造 chosen/rejected
-> DPO 偏好对齐
-> 简历和面试包装
```

## 最小 SQL 学习范围

不需要先成为 SQL 高手，但至少要能看懂：

```text
SELECT
FROM
WHERE
JOIN
GROUP BY
ORDER BY
COUNT / SUM / AVG / MAX / MIN
LIMIT
```

项目重点不是人工写 SQL，而是训练模型根据：

```text
用户问题 + 数据库 schema
```

生成：

```text
可解析、字段正确、逻辑合理、可执行的 SQL
```

## 当前暂定实验设置

T4 低成本路线：

```text
GPU: T4 16G
model: Qwen/Qwen2.5-Coder-7B-Instruct
method: 4bit QLoRA SFT
dataset: text-to-sql-mix-v2 子集
max_samples: 10000 起步
cutoff_len: 1024
epoch: 1
batch_size: 1
gradient_accumulation: 16
learning_rate: 1e-4
lora_rank: 8
lora_alpha: 16
lora_target: q_proj,v_proj
```

如果显存吃紧：

```text
cutoff_len: 768
gradient_accumulation: 32
lora_target 保持 q_proj,v_proj
```
